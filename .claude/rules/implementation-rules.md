# 実装前確認ルール

- コードを書く/編集する全てのタスクにおいて、実装(Edit / Write 等によるコード変更)に着手する前に、設計・実装方針をユーザーに提示し、明示的な承認を得てから実装を開始する
  - 対象はページ・機能追加のような大きなタスクに限らず、1行の修正であっても例外なく適用する
  - 自分(agent)だけで設計内容を判断して実装を進めるのは不可。必ずユーザーの承認を待つ
  - 提示する内容の例: 変更対象のファイル/ディレクトリ構成、主要な設計判断とその理由、テスト方針
  - 実装作業を伴うタスクでは Plan Mode (EnterPlanMode) に入り、ExitPlanMode でユーザーの承認を得てから実装(Edit/Write 等)を開始する
  - 「確認を省略して自律的に進めてよい」という既定動作(auto mode 等)よりも本ルールを優先する
